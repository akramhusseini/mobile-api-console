Mobile API Console

Mirror fetch and XMLHttpRequest traffic from chosen tabs to a local Mobile API Console. Only the patterns you list below will be observed. Saving the form requests the matching host permissions from a user gesture.

Browser profile

This extension tags every event with a stable per-profile id so different browser profiles and incognito contexts stay in separate browser sessions. The id is randomly generated and stored in this profile's extension storage.

How it works

  • Only the background service worker sends events to the console.
  • Page-world capture sees the call bodies. chrome.webRequest supplies response headers when host permissions allow.
  • After updating the unpacked extension files, reload the extension and hard-refresh the page so the latest page-world patch is active.
  • Different browser profiles or regular + incognito produce separate browser sessions in the console.
